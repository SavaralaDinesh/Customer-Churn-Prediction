# Telecom Customer Churn Prediction

### Project Goal
Used Python and Machine Learning to predict telecom customer churn using the IBM Telco dataset. The goal was to understand what factors make customers leave and build a model to identify at-risk customers.

### Tools Used
- **Python**: Pandas for data cleaning, Matplotlib & Seaborn for visualization
- **Machine Learning**: Scikit-learn - Random Forest Classifier
- **Environment**: Jupyter Notebook

### What I Did
1. **Data Cleaning**: Handled missing values in `TotalCharges` column and converted it to numeric. Encoded categorical variables like `Contract` and `InternetService`.
2. **Exploratory Data Analysis**: Used count plots and heatmaps to find that customers with Month-to-Month contracts and high monthly charges had higher churn rates.
3. **Model Building**: Trained a Random Forest model to classify churn vs no-churn.
4. **Model Evaluation**: Checked model performance using accuracy score and confusion matrix.

### Results
- **Model**: Random Forest Classifier
- **Accuracy**: ~85% on test data
- **Key Findings**: Contract type, tenure, and monthly charges were the most important features influencing churn.

### What I Learned
This was my first end-to-end ML project. I learned how to clean real-world data, perform EDA to get insights, and build a classification model. It helped me understand the full data science workflow.
